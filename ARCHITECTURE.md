# معمارية Moataz AI

## الحدود الرئيسية

- `src/pages/`: تجميع الشاشات والمسارات فقط.
- `src/components/`: وحدات الواجهة القابلة لإعادة الاستخدام؛ عناصر الدردشة في `components/chat`.
- `src/lib/`: عملاء API وعقود المتصفح والتحويلات الخالصة.
- `src/features/`: منطق نطاق أكبر عند نمو ميزة مستقلة.
- `shared/`: عقود مشتركة بين المتصفح والخادم، مثل سجل المزودات وأنواع الملفات.
- `api/_handlers/`: واجهات HTTP حسب النطاق، دون أسرار داخل الواجهة.
- `api/_lib/`: المصادقة والتشفير والتحقق والمزودات والسياسات المشتركة.
- `supabase/migrations/`: المصدر المتسلسل لكل تغيير إنتاجي في قاعدة البيانات.

## مسارات البيانات

1. الواجهة تستدعي مسارًا من نفس الأصل عبر `apiJson` مع cookies آمنة.
2. الخادم يصادق الجلسة ويتحقق من الملكية والحدود قبل استخدام Service Role.
3. مفاتيح المزودات لا تعود إلى المتصفح بعد حفظها؛ الخادم يفكها فقط أثناء الطلب الفعلي.
4. المرفقات تُخزن في bucket خاص، ولا يصل المستخدم إليها إلا من `/api/files/:id` بعد فحص الملكية.
5. ملفات المشاريع تبقى نصية ومحدودة الحجم، ويمكن تصديرها كـZIP من الخادم.

## عقد الدردشة

- `ChatProviderControls`: اختيار المزود والنموذج فقط، ولا يقرأ المفاتيح.
- `ChatComposer`: الإدخال، السحب والإفلات، والتحقق الأولي من المرفقات.
- `ChatMessage`: عرض الرسالة والمرفقات وملفات الكود الناتجة.
- `src/lib/chat-api.ts`: محلل SSE؛ يرفض البث الناقص أو الإجابة الفارغة ويحفظ معرّف الطلب للتشخيص.
- `api/chat.ts`: المصادقة، الميزانية الزمنية الكلية، إعادة المحاولة قبل أول مخرجات فقط، وتسوية الاستخدام.
- adapters داخل `api/_lib/providers/`: توحيد اختلافات المزودات دون تسريبها إلى الصفحة.

لا تحفظ الواجهة إجابة مساعد إذا انقطع حدث `done`. إعادة المحاولة تستخدم رسالة المستخدم المحفوظة بدل إنشاء رسالة مكررة.

## إضافة مزود

1. أضف تعريفه مرة واحدة في `shared/provider-registry.ts`.
2. استخدم adapter قائمًا حسب البروتوكول أو أضف adapter معزولًا داخل `api/_lib/providers/`.
3. أضف اختبار endpoint واكتشاف نموذج وتوليد/بث حقيقي.
4. لا تضف Base URL خاصًا بالمزود داخل مكونات الواجهة أو معالج الدردشة.

## إضافة ميزة

ابدأ بعقد مشترك صغير، ثم handler خادمي، ثم عميل في `src/lib`، ثم شاشة/مكوّن. أضف migration قابلة لإعادة التشغيل عند تغيير البيانات، واختبارًا لوظيفة التحقق أو التحويل، ثم نفّذ `npm run check`.
